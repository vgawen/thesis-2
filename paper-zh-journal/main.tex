  \small
  \noindent\textbf{摘\ 要：}大语言模型驱动的端到端 UI 测试自愈方案正被广泛用于降低脆弱定位器的维护成本。本文以 ReproBreak 数据集中 tryghost/koenig 项目的真实 Playwright 失效用例为对象，系统测量该类修复器在缺乏行为级验证机制时的实际行为。在 75 个可达失效用例上，开源代码模型 qwen2.5-coder:7b 于 78.7\% 的对抗性构造场景中给出语法正确但指向错误元素的选择器，构成典型的隐性误修复；附加“无匹配时放弃”的软提示守门规则后结果完全不变，表明提示级守门对该失败模式形同虚设。本文进一步给出 HealReact 系统 L1 静态基底（基于抽象语法树的元素抽取器、意图标注器与确定性校准）的实现，可静态覆盖 80.6\% 的真实失效目标；其上的 L3 修复器在可达用例上的 top-1 正确率为 77.3\%。实验结果表明：在无人值守的持续集成自愈场景下，引入行为重放验证器是必要的下一步。

  \vspace{2pt}
  \noindent\textbf{关键词：}软件测试；测试自愈；大语言模型；Playwright；React；抽象语法树

  \vspace{2pt}
  \noindent\textbf{中图分类号：}TP311.5\quad\textbf{文献标识码：}A

  \vspace{4pt}
  \noindent\textbf{Abstract:} LLM-driven self-healing for end-to-end UI tests is increasingly used to absorb the maintenance cost of brittle locators. On real-world Playwright break cases from the ReproBreak dataset, across 75 reachable koenig breaks, qwen2.5-coder:7b silently emits a wrong yet resolver-valid Playwright selector in 78.7\% of adversarial-by-construction cases; an explicit ``abstain'' prompt guardrail produces identical numbers. HealReact's L1 substrate (AST extractor + intent labeller + deterministic calibration) reaches 80.6\% of the 93 real break targets statically; on top of it the L3 healer answers 77.3\% of reachable breaks correctly. We make the empirical case that a behavioural-replay oracle is a necessary next layer for unattended CI healing.

  \vspace{2pt}
  \noindent\textbf{Keywords:} software testing; self-healing test; large language model; Playwright; React; abstract syntax tree
