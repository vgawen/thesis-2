  \small
  \noindent\textbf{摘\ 要：}大语言模型驱动的 UI 测试自愈正用于降低脆弱定位器的维护成本。本文在 ReproBreak 数据集 koenig 项目 75 个真实 Playwright 可达失效用例上，于\emph{选择器解析层面}测量代码模型 qwen2.5-coder:7b 在缺乏行为验证时的行为：在对抗性构造场景下，78.7\% 的用例中模型给出可解析但指向错误元素的选择器（即选择器层面隐性误修复的对抗性上界），附加“无匹配则放弃”软提示后结果不变。给出 HealReact 系统 L1 静态基底（AST 抽取、意图标注、确定性校准），静态覆盖 80.6\% 真实失效目标，L3 修复器在可达用例上 top-1 选择器精确匹配率达 77.3\%。结果表明：无人值守持续集成自愈场景下，引入行为重放验证器是必要的下一步；本文数字为静态解析层面的诊断性指标，与运行时测试通过率之间的换算仍需真实重放。

  \vspace{2pt}
  \noindent\textbf{关键词：}测试自愈；大语言模型；Playwright；React；隐性误修复

  \vspace{2pt}
  \noindent\textbf{中图分类号：}TP311.5\quad\textbf{文献标识码：}A

  \vspace{4pt}
  \noindent\textbf{Abstract:} LLM-driven self-healing for end-to-end UI tests is increasingly used to absorb the maintenance cost of brittle locators. On real-world Playwright break cases from the ReproBreak dataset, across 75 reachable koenig breaks, we measure -- at the selector-resolution level -- that qwen2.5-coder:7b emits a wrong yet resolver-valid Playwright selector in 78.7\% of adversarial-by-construction cases (an adversarial upper bound on selector-level silent false healing); an explicit ``abstain'' prompt guardrail produces identical numbers. HealReact's L1 substrate (AST extractor + intent labeller + deterministic calibration) reaches 80.6\% of the 93 real break targets statically; on top of it the L3 healer attains 77.3\% top-1 exact-element selector matching on reachable breaks. We make the empirical case that a behavioural-replay oracle is a necessary next layer for unattended CI healing; all numbers here are static, selector-resolution diagnostics, and their mapping to runtime test-pass rates still requires real replay.

  \vspace{2pt}
  \noindent\textbf{Keywords:} software testing; self-healing test; large language model; Playwright; React; abstract syntax tree

  \vspace{6pt}
